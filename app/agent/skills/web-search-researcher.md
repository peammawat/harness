# ทักษะการค้นหาข้อมูลบนเว็บ

เมื่อต้องตอบคำถามที่อาศัยข้อมูลจากภายนอก ให้ค้นคว้าอย่างเป็นระบบตามแนวทางนี้ เป้าหมายคือคำตอบที่ทั้งถูกต้องและเป็นปัจจุบัน พร้อมแหล่งอ้างอิงที่ตรวจสอบได้

> หมายเหตุ: ทักษะนี้เป็น "วิธีค้น" ไม่ใช่ตัวค้นเอง ต้องเปิดความสามารถค้นหา (Web Search ในตัว หรือ SearXNG tool) ไว้ด้วยจึงจะค้นได้จริง

## 1. เมื่อไหร่ที่ต้องค้น

**ค้นเมื่อ:**
- ข้อมูลที่เปลี่ยนบ่อย: ราคา (ทอง น้ำมัน หุ้น คริปโต อัตราแลกเปลี่ยน) สภาพอากาศ ข่าว เหตุการณ์ล่าสุด
- ข้อเท็จจริงที่อาจล้าสมัย: ใครดำรงตำแหน่งอะไรตอนนี้ เวอร์ชันล่าสุดของซอฟต์แวร์/ไลบรารี กฎหมายหรือนโยบายปัจจุบัน
- เรื่องที่อยู่นอกความรู้ หรือเมื่อไม่แน่ใจในความถูกต้อง
- ผู้ใช้ขอให้ค้นหรืออ้างอิงแหล่งโดยตรง

**ไม่ต้องค้นเมื่อ:** ความรู้ทั่วไปที่นิ่ง (คำนิยาม หลักการ คณิตศาสตร์ ประวัติศาสตร์ที่จบแล้ว) หรือคำถามเชิงความเห็นและงานสร้างสรรค์

## 2. การตั้งคำค้น (query)

- สั้น กระชับ เจาะจง 3–6 คำ เริ่มกว้างก่อนแล้วค่อยแคบลง
- คำค้นแต่ละครั้งต้องต่างกันจริง ถ้าครั้งแรกไม่เจอ ให้เปลี่ยนคำ/มุมมอง/แหล่ง อย่ายิงคำเดิมซ้ำ
- เรื่องเกี่ยวกับไทยให้ค้นเป็นภาษาไทย (เช่น `ราคาทองวันนี้`) ส่วนเรื่องสากลหรือเทคนิคค้นเป็นภาษาอังกฤษ
- ใส่ปี/วันที่เมื่อต้องการข้อมูลตามช่วงเวลา และใช้ปีปัจจุบันจริงเสมอ — ระวังการสลับ พ.ศ./ค.ศ. (เช่น ค.ศ. 2026 = พ.ศ. 2569) อย่าใส่ปีเก่าเพราะจะได้ผลลัพธ์ล้าสมัย
- คำถามที่มีหลายส่วนหรือหลายรายการ ให้แยกค้นทีละรายการ อย่ารวบเป็นคำค้นเดียวเพราะจะได้ผลตื้นสำหรับทุกอย่าง

## 3. จำนวนครั้งที่ค้น (ปรับตามความซับซ้อน)

- ข้อเท็จจริงเดียว: 1 ครั้งพอ
- เปรียบเทียบหรือมีหลายแง่มุม: 3–8 ครั้ง
- งานวิจัยเชิงลึกหรือคำถามปลายเปิด: มากกว่านั้นตามความจำเป็น
- ค้นต่อไปจนกว่าทุกส่วนของคำตอบจะมีแหล่งรองรับ อย่าหยุดกลางคันแล้วเดาส่วนที่เหลือ

## 4. อ่านเนื้อหาฉบับเต็มเมื่อจำเป็น

snippet จากผลค้นหามักสั้นเกินกว่าจะเชื่อถือได้ ถ้าต้องการตัวเลข ราคา คำพูดอ้างอิง หรือรายละเอียดเฉพาะ ให้เปิดอ่านหน้าเว็บฉบับเต็มก่อนสรุป อย่าสรุปจากหัวข้อหรือคำโปรยอย่างเดียว

## 5. ประเมินความน่าเชื่อถือของแหล่ง

- เลือกแหล่งต้นทางหรือแหล่งทางการก่อน: เว็บหน่วยงานรัฐ งานวิจัย เอกสารทางการ หรือเจ้าของข้อมูลโดยตรง (เช่น ราคาทอง → สมาคมค้าทองคำ, ข้อมูลซอฟต์แวร์ → เว็บทางการ/release notes)
- ให้น้ำหนักกับความใหม่ เรื่องที่เปลี่ยนเร็วให้เน้นแหล่งล่าสุด
- ตรวจสอบข้าม (cross-check) อย่างน้อย 2 แหล่งเมื่อข้อมูลสำคัญหรือดูขัดแย้งกัน
- ระวังเป็นพิเศษกับ: ฟอรัม เนื้อหาที่ทำ SEO หนัก ๆ รีวิวสินค้า และแหล่งที่อาจมีผลประโยชน์แอบแฝง
- ถ้าแหล่งให้ข้อมูลไม่ตรงกัน ให้บอกผู้ใช้ตามตรงว่าข้อมูลขัดแย้งกัน พร้อมระบุว่าแต่ละแหล่งว่าอย่างไร แทนที่จะเลือกข้างเอง

## 6. การอ้างอิง

- อ้างอิงแหล่งที่มาเสมอเมื่อใช้ข้อมูลจากเว็บ พร้อมลิงก์ URL ที่เปิดได้
- อ้างเฉพาะแหล่งที่มีผลต่อคำตอบจริง ไม่ต้องยัดทุกแหล่งที่เจอ
- อย่ากุชื่อแหล่งหรือ URL เด็ดขาด ถ้าไม่แน่ใจในข้อมูลใด ให้ตัดข้อความนั้นออก

## 7. การเรียบเรียงคำตอบ

- ตอบให้ตรงคำถามก่อน วางข้อมูลล่าสุดและสำคัญที่สุดไว้ต้น ๆ
- กระชับ ไม่ซ้ำซาก ใส่เฉพาะข้อมูลที่เกี่ยวข้องกับคำถาม
- ระบุวันที่หรือเวลาอ้างอิงของข้อมูลเมื่อเกี่ยวข้อง (เช่น "ราคา ณ 09:00 น. วันที่ ...") เพื่อให้ผู้ใช้รู้ว่าข้อมูลสดแค่ไหน
- ถ้าค้นแล้วไม่พบข้อมูลที่เชื่อถือได้ ให้บอกตามตรง อย่าเดาหรือแต่งคำตอบ

## 8. การจัดรูปแบบคำตอบ (Markdown)

ตอบเป็น Markdown ให้อ่านง่ายและสวยงาม โดยเลือกใช้เท่าที่จำเป็นกับความยาวของคำตอบ:

- คำตอบสั้น ๆ ตอบเป็นข้อความหรือ bullet ธรรมดา ไม่ต้องใส่หัวข้อให้รก
- คำตอบยาวหลายประเด็น ใช้หัวข้อ `##` / `###` แบ่งส่วน — **เว้นวรรคหลัง `#` เสมอ** (เช่น `## สรุป` ไม่ใช่ `##สรุป`)
- เน้นตัวเลข ราคา หรือข้อสรุปสำคัญด้วย **ตัวหนา** ให้สะดุดตา
- ใช้ bullet (`-`) สำหรับรายการ และใช้ **ตาราง** เมื่อเปรียบเทียบหลายตัวเลือกหรือหลายมิติ (เช่น เทียบสเปก เทียบราคา)
- ปิดท้ายด้วยส่วน **แหล่งอ้างอิง** รวมลิงก์ที่ใช้จริงตามกฎในข้อ 6
- อย่าใส่ element เกินจำเป็น — รูปแบบมีไว้ช่วยให้อ่านง่าย ไม่ใช่ตกแต่งให้รก
